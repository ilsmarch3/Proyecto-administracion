Aquí tienes una guía completa para usar **GPG (GNU Privacy Guard)** desde la terminal. Cubre desde la instalación hasta la gestión de llaves, encriptación y desencriptación.

---

### 1. Instalación y Primeros Pasos

Antes de empezar, necesitas tener GPG instalado.

- **En Linux (Debian/Ubuntu):**
    ```bash
    sudo apt install gpg
    ```
    

- **En macOS:**
    ```bash
    brew install gnupg
    ```

- **En Windows:** Descarga e instala **Gpg4win** desde su sitio oficial.

Para verificar que todo está funcionando, puedes consultar la ayuda:
```bash
gpg --help
```
Este comando mostrará la versión instalada y los algoritmos de cifrado disponibles .

---

### 2. Creación de Llaves (Pública y Privada)

El corazón de GPG es el **par de llaves**. La *pública* se comparte para que te envíen archivos cifrados, y la *privada* es secreta y se usa para desencriptar.

Para generar un par nuevo, ejecuta:
```bash
gpg --full-generate-key
```


El asistente te preguntará varias cosas:
1.  **Tipo de llave:** Generalmente selecciona `1` (RSA y RSA) o `9` (ECC).
2.  **Tamaño:** Si elegiste RSA, escribe `4096` (más seguro) .
3.  **Caducidad:** Si pones `0` la llave nunca expira .
4.  **Identidad:** Es importante que pongas tu nombre real y tu correo electrónico .
5.  **Frase de contraseña (Passphrase):** Es una capa extra de seguridad para tu llave privada. **No la olvides**.

### 3. Gestión de Llaves (Listar, Exportar, Importar)

Una vez creadas las llaves, necesitarás saber cuáles tienes, compartir la pública e importar las de otros.

**Listar llaves**
- **Públicas:** `gpg --list-keys` (o `gpg -k`). Te muestra la lista de llaves que tienes guardadas de otros y la tuya .
- **Privadas (Secretas):** `gpg --list-secret-keys` (o `gpg -K`). Muestra las llaves que realmente te pertenecen .

**Exportar Llaves**
- **Tu Llave Pública (para compartir):** Esto genera un archivo de texto que puedes enviar por email o subir a un servidor.
    ```bash
    gpg --export -a "Tu Nombre" > mi_llave_publica.asc
    ```
    El flag `-a` (o `--armor`) genera un texto legible en vez de datos binarios .
- **Tu Llave Privada (Backup):** Guárdala en un lugar seguro (como un USB cifrado). **Nunca la compartas**.
    ```bash
    gpg --export-secret-key -a "Tu Nombre" > mi_llave_privada_backup.asc
    ```
    

**Importar Llaves**
Cuando alguien te envía su llave pública (archivo `.asc`), la guardas en tu llavero así:
```bash
gpg --import archivo_llave_publica.asc
```


### 4. Encriptar Archivos

Para enviar un archivo seguro a alguien (llamémosle "Ana"), necesitas **su llave pública** (debes haberla importado antes con `--import`).

- **Encriptar (solo para Ana):**
    ```bash
    gpg --encrypt --recipient "ana@email.com" archivo_secreto.txt
    ```
    Esto generará un archivo llamado `archivo_secreto.txt.gpg`.
    *Nota:* Asegúrate de que el nombre (o email) coincida exactamente con el UID que aparece en `gpg --list-keys` .

- **Encriptar (usando ID):**
    Puedes usar el ID de la llave (la cadena hexadecimal) si es más fácil:
    ```bash
    gpg -e -r FE0BC1AD74A10B23 archivo.txt
    ```
    

### 5. Desencriptar Archivos

Cuando *tú* recibes un archivo cifrado para ti (`.gpg`), usas tu **llave privada**.

- **Desencriptar a archivo original:**
    ```bash
    gpg --output documento_original.txt --decrypt archivo_cifrado.gpg
    ```
    Te pedirá tu "passphrase" (la contraseña de tu llave privada) y generará el archivo legible .

- **Ver en consola (sin guardar):**
    Si solo quieres ver el contenido sin guardarlo:
    ```bash
    gpg --decrypt archivo_cifrado.gpg
    ```
    

### 6. Firmar y Verificar

La firma garantiza que el archivo proviene de ti y no ha sido modificado.

- **Firmar un archivo (crea un archivo `.sig`):**
    ```bash
    gpg --sign documento.txt
    ```
    Esto crea `documento.txt.gpg` en formato binario.

- **Firma Clearsign (texto legible):**
    Si quieres firmar un texto pero que siga siendo legible para humanos (útil en correos):
    ```bash
    gpg --clearsign documento.txt
    ```
    Esto crea `documento.txt.asc`.

- **Verificar firma:**
    Si recibes un archivo y una firma, verificas quién lo envió.
    ```bash
    gpg --verify documento.txt.gpg
    ```
    

### 7. Comandos Básicos de Rescate

| Acción | Comando | Explicación |
| :--- | :--- | :--- |
| **Eliminar Llave Pública** | `gpg --delete-key "Usuario"` | Elimina la llave pública de tu llavero . |
| **Eliminar Llave Privada** | `gpg --delete-secret-key "Usuario"` | Elimina tu llave privada (cuidado) . |
| **Buscar en Servidores** | `gpg --keyserver keyserver.ubuntu.com --search-keys "email@dominio.com"` | Busca llaves públicas en internet . |
| **Exportar Confianza** | `gpg --export-ownertrust > confianza.txt` | Útil para migrar la configuración de confianza a otra PC . |

### 8. Mejores Prácticas de Seguridad

1.  **Usa `--batch` en scripts:** Si automatizas procesos, usa los flags `--batch --yes` para evitar que el sistema espere tu interacción .
2.  **Tamaño de llave:** Siempre usa **RSA 4096** bits o superior .
3.  **Copia de seguridad:** Guarda tu llave privada (`--export-secret-key`) en un lugar físico seguro (como un disco duro externo apagado).
4.  **Revocación:** Al crear una llave, considera generar un *certificado de revocación* (`gpg --gen-revoke`) por si pierdes el control de tu llave privada .
